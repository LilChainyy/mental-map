# API Reference

Mental Map exposes two server-side API routes via Vite's API route system. Both are thin proxies to the Anthropic Claude API and return structured JSON. They are accessible only during `npm run dev` or `npm run preview` — this is not a publicly deployed API.

---

## Authentication

Both routes require `ANTHROPIC_API_KEY` to be set in the server environment (`.env.local`). The key is never exposed to the browser. Requests to these routes from the frontend do not require any client-side auth.

---

## POST /api/generate-graph

Generates a concept knowledge graph for a given search term.

### Request

```
POST /api/generate-graph
Content-Type: application/json
```

```json
{
  "concept": "string"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `concept` | string | Yes | The concept to generate a graph for (e.g. `"transformer architecture"`, `"Black-Scholes model"`, `"tort law"`) |

### Response — 200 OK

```json
{
  "centerNodeId": "string",
  "nodes": [
    {
      "id": "string",
      "label": "string",
      "description": "string"
    }
  ],
  "edges": [
    {
      "source": "string",
      "target": "string",
      "color": "blue" | "red"
    }
  ]
}
```

| Field | Type | Description |
|---|---|---|
| `centerNodeId` | string | The `id` of the node representing the searched concept. Must match one of the node IDs in `nodes`. |
| `nodes` | array | 8–15 concept nodes. Each has a kebab-case `id`, a human-readable `label`, and a 1–2 sentence `description`. |
| `edges` | array | Connections between nodes. `source` and `target` are node IDs. `color: "blue"` = related/complementary; `color: "red"` = contrasting/conflicting. |

### Response — 400 Bad Request

```json
{ "error": "Missing or invalid \"concept\" field" }
```

### Response — 405 Method Not Allowed

```json
{ "error": "Method not allowed" }
```

### Response — 500 Internal Server Error

```json
{ "error": "string" }
```

Covers Claude API errors, JSON parse failures, and unexpected exceptions.

### Notes

- The current system prompt restricts generated concepts to CS/software engineering. This is a known limitation tracked in [ROADMAP.md](ROADMAP.md).
- Node IDs are kebab-case slugs (e.g. `"attention-mechanism"`, `"gradient-descent"`). The center node's ID is the kebab-case of the input concept.
- The client (`useGenerateGraph.ts`) caches responses in IndexedDB by normalized concept key. Repeated calls with the same concept string skip this endpoint entirely.

## POST /api/stitch-edges

Analyzes disconnected concept clusters on the current map and suggests cross-cluster connections. Optionally synthesizes "bridge nodes" — new concepts that naturally link distant clusters.

### Request

```
POST /api/stitch-edges
Content-Type: application/json
```

```json
{
  "nodes": ["string"],
  "existingEdges": [
    { "source": "string", "target": "string" }
  ],
  "clusters": [["string"]]
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `nodes` | string[] | Yes | Array of node **labels** (not IDs) for all nodes currently on the map |
| `existingEdges` | array | Yes | Array of `{ source, target }` pairs using node labels, representing all current edges |
| `clusters` | string[][] | No | Pre-computed cluster groupings (each sub-array is a cluster of node labels). When provided and contains >1 cluster, Claude is prompted to connect all clusters. When omitted, Claude finds missing connections across all nodes. |

### Response — 200 OK

```json
{
  "edges": [
    {
      "source": "string",
      "target": "string",
      "color": "blue" | "red",
      "reason": "string"
    }
  ],
  "bridgeNodes": [
    {
      "id": "string",
      "label": "string",
      "description": "string"
    }
  ]
}
```

| Field | Type | Description |
|---|---|---|
| `edges` | array | Suggested cross-cluster edges. `source` and `target` are node **labels** (matching `nodes` input or bridge node labels). `reason` is a 1-sentence explanation of the relationship. Max 12 edges. |
| `bridgeNodes` | array | Optional synthesized bridge concepts. Each has a kebab-case `id`, `label`, and `description`. Max 3 bridge nodes. Edges referencing bridge nodes use the bridge node's label as source or target. |

### Response — 400 Bad Request

```json
{ "error": "Missing or invalid \"nodes\" field" }
```

### Response — 405 Method Not Allowed

```json
{ "error": "Method not allowed" }
```

### Response — 500 Internal Server Error

```json
{ "error": "string" }
```

### Notes

- The stitch API uses **node labels** (not IDs) for source/target references, including in bridge node edges. The client resolves labels back to IDs after receiving the response.
- Bridge node IDs are kebab-case slugs generated by Claude. The client creates actual `GraphNode` records with `isBridgeNode: true`.
- The client (`useStitchEdges.ts`) filters out any returned edges whose `source|target` key appears in the `deletedStitchedEdges` table — i.e., edges the user has previously deleted will not be re-created.
- The current system prompt restricts stitching to CS concepts. This will be updated alongside the generation prompt (see ROADMAP.md).

---

## Error Handling

Both endpoints follow the same error shape: `{ "error": "string" }`. The client surfaces errors via `alert()` in the current implementation. Error handling improvements (toast notifications, inline error states) are tracked in the roadmap.

---

## Rate Limiting

There is no server-side rate limiting. The Anthropic API enforces its own rate limits at the key level. Heavy interactive use (rapid searching or repeated stitching) can hit rate limits; the client will receive a 500 with the Anthropic error message propagated through.
